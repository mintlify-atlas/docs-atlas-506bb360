A data provider extension is a Python package that implements the `Fetcher` interface and registers itself with the platform via an entry point. Once installed, the provider becomes available as a `provider` choice on every command that uses the same standard model.

## The TET pattern

Every `Fetcher` follows the **Transform → Extract → Transform** pipeline:

1. **Transform** (`transform_query`) — Convert the raw input parameters into a validated `QueryParams` model. This is where you rename fields, set defaults, or coerce types.
2. **Extract** (`extract_data` / `aextract_data`) — Use the validated query to call the upstream API and return the raw response (a dict, list, or DataFrame).
3. **Transform** (`transform_data`) — Map the raw response into a list of validated `Data` model instances.

The platform calls these three methods in sequence every time a user runs a command backed by your fetcher.

## Step 1: Create the Python package

Create a new directory with the following structure:

```
openbb_my_provider/
├── pyproject.toml
├── openbb_my_provider/
│   ├── __init__.py
│   └── models/
│       ├── __init__.py
│       └── equity_historical.py
```

In `pyproject.toml`, declare `openbb-provider` as a dependency and register the entry point:

```toml pyproject.toml
[tool.poetry]
name = "openbb-my-provider"
version = "0.1.0"
description = "My custom OpenBB data provider."
readme = "README.md"
packages = [{ include = "openbb_my_provider" }]

[tool.poetry.dependencies]
python = "^3.9"
openbb-core = "^1.0.0"

[tool.poetry.plugins."openbb_provider_extension"]
my_provider = "openbb_my_provider:my_provider"

[build-system]
requires = ["poetry-core"]
build-backend = "poetry.core.masonry.api"
```

<Tip>
Use a loose version range for `openbb-core` (e.g. `>=1.0,<2`) to avoid unnecessary conflicts when users update the platform.
</Tip>

## Step 2: Define the QueryParams model

`QueryParams` holds the validated input parameters that will be forwarded to the upstream API. Inherit from the relevant standard model when one exists — this makes your provider a drop-in replacement on existing commands and enables automatic parameter documentation.

```python openbb_my_provider/models/equity_historical.py
from typing import Optional
from datetime import date

from openbb_core.provider.abstract.query_params import QueryParams
from openbb_core.provider.standard_models.equity_historical import (
    EquityHistoricalQueryParams,
)
from pydantic import Field


class MyProviderEquityHistoricalQueryParams(EquityHistoricalQueryParams):
    """MyProvider equity historical query parameters.

    Source: https://www.myprovider.com/documentation/
    """

    # Add provider-specific parameters below the standard ones.
    # Standard fields (symbol, start_date, end_date) are inherited automatically.
    adjusted: Optional[bool] = Field(
        default=True,
        description="Return adjusted OHLCV data.",
    )
```

If no standard model exists for your data type, inherit directly from `QueryParams`:

```python openbb_my_provider/models/custom_data.py
from openbb_core.provider.abstract.query_params import QueryParams
from pydantic import Field


class MyProviderCustomQueryParams(QueryParams):
    """MyProvider custom data query parameters."""

    symbol: str = Field(description="Ticker symbol.")
    limit: int = Field(default=100, description="Number of records to return.")
```

## Step 3: Define the Data model

`Data` holds the validated output records. Inherit from the matching standard model so that your output is compatible with downstream analytics extensions (charting, technical analysis, etc.).

```python openbb_my_provider/models/equity_historical.py
from typing import Optional
from datetime import datetime

from openbb_core.provider.abstract.data import Data
from openbb_core.provider.standard_models.equity_historical import (
    EquityHistoricalData,
)
from pydantic import Field


class MyProviderEquityHistoricalData(EquityHistoricalData):
    """MyProvider equity historical OHLCV data.

    Source: https://www.myprovider.com/documentation/
    """

    # Add provider-specific fields that are not part of the standard model.
    adj_close: Optional[float] = Field(
        default=None,
        description="Adjusted closing price.",
    )
    split_factor: Optional[float] = Field(
        default=None,
        description="Split adjustment factor.",
    )
```

<Note>
The `Data` class uses Pydantic's `extra="allow"` configuration, so any extra fields returned by the API are preserved and accessible on the result objects — you only need to declare fields explicitly when you want type coercion, validation, or documentation.
</Note>

## Step 4: Build the Fetcher class

The `Fetcher` is a generic class parameterized by your `QueryParams` and `Data` types. Implement the three TET methods as `@staticmethod`.

```python openbb_my_provider/models/equity_historical.py
import asyncio
from typing import Any, Dict, List, Optional

import httpx
from openbb_core.provider.abstract.fetcher import Fetcher

from openbb_my_provider.models.equity_historical import (
    MyProviderEquityHistoricalQueryParams,
    MyProviderEquityHistoricalData,
)


class MyProviderEquityHistoricalFetcher(
    Fetcher[
        MyProviderEquityHistoricalQueryParams,
        List[MyProviderEquityHistoricalData],
    ]
):
    """Transform the query, extract, and transform equity historical data."""

    @staticmethod
    def transform_query(
        params: Dict[str, Any]
    ) -> MyProviderEquityHistoricalQueryParams:
        """Transform raw input parameters into the validated query model."""
        return MyProviderEquityHistoricalQueryParams(**params)

    @staticmethod
    async def aextract_data(
        query: MyProviderEquityHistoricalQueryParams,
        credentials: Optional[Dict[str, str]],
        **kwargs: Any,
    ) -> List[Dict]:
        """Fetch raw data from the MyProvider API."""
        api_key = credentials.get("my_provider_api_key") if credentials else None

        params = {
            "symbol": query.symbol,
            "from": query.start_date,
            "to": query.end_date,
            "adjusted": query.adjusted,
            "apikey": api_key,
        }

        async with httpx.AsyncClient() as client:
            response = await client.get(
                "https://api.myprovider.com/v1/historical",
                params={k: v for k, v in params.items() if v is not None},
            )
            response.raise_for_status()
            return response.json().get("data", [])

    @staticmethod
    def transform_data(
        query: MyProviderEquityHistoricalQueryParams,
        data: List[Dict],
        **kwargs: Any,
    ) -> List[MyProviderEquityHistoricalData]:
        """Map raw API records to the validated data model."""
        return [MyProviderEquityHistoricalData.model_validate(record) for record in data]
```

<Tip>
Use `aextract_data` (the async variant) when calling an external HTTP API. The platform awaits it automatically. Use the synchronous `extract_data` only when the data source does not support async (e.g. a local file or synchronous SDK).
</Tip>

### Optional credentials

By default, the platform raises an error before running a fetcher if any declared credential is missing. To allow the fetcher to run without credentials (for example when the API has a free unauthenticated tier), set `require_credentials = False` on the class:

```python
class MyProviderEquityHistoricalFetcher(
    Fetcher[
        MyProviderEquityHistoricalQueryParams,
        List[MyProviderEquityHistoricalData],
    ]
):
    require_credentials = False

    ...
```

### Field name mapping

If the API returns field names that differ from your `Data` model, use `__alias_dict__` to map them:

```python
class MyProviderEquityHistoricalData(EquityHistoricalData):
    """MyProvider equity historical data."""

    # Map API field name → model field name
    __alias_dict__ = {
        "o": "open",
        "h": "high",
        "l": "low",
        "c": "close",
        "v": "volume",
    }
```

## Step 5: Define the Provider instance

The `Provider` object ties your fetchers together and declares any credentials the provider requires. Credential names are automatically prefixed with `{provider_name}_` — so `"api_key"` becomes `"my_provider_api_key"` in the user's credentials store.

```python openbb_my_provider/__init__.py
"""MyProvider provider module."""

from openbb_core.provider.abstract.provider import Provider

from openbb_my_provider.models.equity_historical import (
    MyProviderEquityHistoricalFetcher,
)

my_provider = Provider(
    name="my_provider",
    website="https://www.myprovider.com",
    description="Financial data from MyProvider.",
    credentials=["api_key"],  # stored as "my_provider_api_key"
    fetcher_dict={
        # Key must match the standard model name used by the command's model= argument
        "EquityHistorical": MyProviderEquityHistoricalFetcher,
    },
)
```

If your provider requires no credentials (public API), omit the `credentials` parameter.

## Step 6: Register the entry point

The entry point in `pyproject.toml` tells the platform where to find the `Provider` instance. The format is:

```
<arbitrary_name> = "<module.path>:<provider_attribute>"
```

```toml pyproject.toml
[tool.poetry.plugins."openbb_provider_extension"]
my_provider = "openbb_my_provider:my_provider"
```

Install the package so the entry point is registered:

```bash
pip install -e .
```

Then rebuild the OpenBB package:

```bash
python -c "import openbb; openbb.build()"
```

Your provider now appears as a `provider` choice on every command backed by a model in your `fetcher_dict`:

```python
from openbb import obb

result = obb.equity.price.historical("AAPL", provider="my_provider")
```

## Testing the fetcher

Every `Fetcher` class ships with a `test` classmethod that runs each TET stage independently and asserts the output types are correct. Call it from a test file:

```python tests/test_equity_historical.py
import pytest
from openbb_my_provider.models.equity_historical import (
    MyProviderEquityHistoricalFetcher,
)


@pytest.mark.record_http
def test_my_provider_equity_historical_fetcher(credentials=None):
    params = {
        "symbol": "AAPL",
        "start_date": "2023-01-01",
        "end_date": "2023-01-31",
    }

    fetcher = MyProviderEquityHistoricalFetcher()
    fetcher.test(params=params, credentials=credentials)
```

Run with recorded HTTP responses so tests stay deterministic:

```bash
# Record real HTTP responses on first run
pytest tests/test_equity_historical.py --record=all

# Replay recordings on subsequent runs (no network required)
pytest tests/test_equity_historical.py
```

<Warning>
The `test` method asserts that `extract_data` returns **raw** (non-transformed) records. Do not call `model_validate` inside `extract_data` — transformation belongs in `transform_data`.
</Warning>
