A provider is a self-contained package that connects a financial data source to the OpenBB standardization layer. Each provider implements one or more `Fetcher` classes that handle authentication, HTTP requests, and data transformation — all behind a uniform interface.

From your perspective, switching providers is a single keyword argument:

```python
from openbb import obb

# Yahoo Finance (free, no key required)
obb.equity.price.historical("AAPL", provider="yfinance")

# Financial Modeling Prep (requires API key)
obb.equity.price.historical("AAPL", provider="fmp")

# Intrinio (requires API key)
obb.equity.price.historical("AAPL", provider="intrinio")
```

The returned `OBBject` structure is identical regardless of provider. Only the fields inside `results` — and any provider-specific `extra_params` — differ.

## Selecting a provider

Pass `provider` as a keyword argument to any command. If you omit it, OpenBB uses the first provider in its resolution order that has valid credentials.

```python
from openbb import obb

# Explicit provider selection
result = obb.equity.price.historical("AAPL", provider="fmp")
print(result.provider)  # "fmp"

# Omit provider — OpenBB resolves automatically
result = obb.equity.price.historical("AAPL")
print(result.provider)  # whichever provider resolved first
```

## Multiple providers for the same endpoint

Many endpoints are implemented by several providers. You can query each in sequence to compare or fall back:

```python
from openbb import obb

providers = ["fmp", "intrinio", "yfinance"]

for p in providers:
    try:
        result = obb.equity.price.historical("AAPL", provider=p)
        print(f"{p}: {len(result.results)} rows")
    except Exception as e:
        print(f"{p} failed: {e}")
```

<Tip>
  Use `result.provider` to log which provider actually served each response — especially useful when the provider argument is omitted and OpenBB resolves automatically.
</Tip>

## Provider-specific parameters

Standard parameters (like `symbol`, `start_date`, `end_date`) work across all providers that implement an endpoint. Providers can also expose extra parameters that only they support.

```python
from openbb import obb

# FMP-specific: interval and adjustment type
result = obb.equity.price.historical(
    "AAPL",
    provider="fmp",
    interval="1h",                       # FMP extra param
    adjustment="splits_and_dividends",   # FMP extra param
)
```

Extra parameters are validated by the provider's `QueryParams` model. Passing an unsupported extra parameter to a different provider is silently ignored.

## Extension types

OpenBB providers are packaged as Python extensions in three tiers:

<AccordionGroup>
  <Accordion title="OpenBB extensions">
    Built and maintained by the OpenBB team. Included in the default `openbb` package (e.g., `openbb-fmp`, `openbb-yfinance`, `openbb-fred`). Updated with each OpenBB release.
  </Accordion>
  <Accordion title="Community extensions">
    Built by the community, primarily maintained by OpenBB. Available as optional extras (e.g., `openbb-nasdaq`, `openbb-cboe`). Reviewed by OpenBB before inclusion.
  </Accordion>
  <Accordion title="Independent extensions">
    Built and maintained by third parties. Published to PyPI independently. Follow the same `Provider` + `Fetcher` pattern but are not bundled with OpenBB.
  </Accordion>
</AccordionGroup>

## How providers work internally

Each provider package defines a `Provider` instance and registers one or more `Fetcher` classes against data model keys:

```python
from openbb_core.provider.abstract.provider import Provider

fmp_provider = Provider(
    name="fmp",
    website="https://financialmodelingprep.com",
    description="Financial Modeling Prep data provider.",
    credentials=["api_key"],   # becomes "fmp_api_key" in user settings
    fetcher_dict={
        "EquityHistorical": FMPEquityHistoricalFetcher,
        "EquityQuote": FMPEquityQuoteFetcher,
        # ...
    },
    repr_name="Financial Modeling Prep (FMP)",
)
```

The `credentials` list entries are automatically namespaced: `["api_key"]` becomes `fmp_api_key` in `user_settings.json`. See [Credentials](/concepts/credentials) for how to set them.
