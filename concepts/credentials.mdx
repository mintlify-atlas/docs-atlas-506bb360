Many OpenBB providers require an API key to access their data. Credentials are loaded at startup, validated as Pydantic `SecretStr` values, and made available across all commands without any per-request configuration.

## Where credentials are stored

OpenBB reads credentials from `~/.openbb_platform/user_settings.json`. This file is created automatically when you first run OpenBB. If it does not exist, OpenBB falls back to environment variables.

```
~/.openbb_platform/
└── user_settings.json
```

## Format of `user_settings.json`

The file is standard JSON. Credentials live inside a top-level `credentials` object. Keys follow the pattern `<provider_name>_<credential_name>`.

```json
{
  "credentials": {
    "fmp_api_key": "YOUR_FMP_KEY",
    "intrinio_api_key": "YOUR_INTRINIO_KEY",
    "fred_api_key": "YOUR_FRED_KEY",
    "tiingo_token": "YOUR_TIINGO_TOKEN",
    "benzinga_api_key": "YOUR_BENZINGA_KEY",
    "tradingeconomics_api_key": "YOUR_TE_KEY",
    "alpha_vantage_api_key": "YOUR_AV_KEY",
    "nasdaq_api_key": "YOUR_NASDAQ_KEY",
    "tradier_api_key": "YOUR_TRADIER_KEY",
    "econdb_api_key": "YOUR_ECONDB_KEY"
  }
}
```

<Note>
  The file may also contain `preferences` and `defaults` sections. The `credentials` key is the only one relevant to API authentication.
</Note>

## Setting credentials in Python

You can set or update credentials at runtime without editing the file. Use `obb.user.credentials`:

```python
from openbb import obb

# Set a single credential
obb.user.credentials.fmp_api_key = "YOUR_FMP_KEY"

# Inspect all configured credentials (values are masked)
print(obb.user.credentials)
# Credentials
#
# fmp_api_key: **********
# fred_api_key: **********
# intrinio_api_key: None
# ...

# Unmask and print all credential values
obb.user.credentials.show()
```

<Warning>
  Credentials set at runtime are not persisted back to `user_settings.json`. They apply only for the current Python session. Edit the JSON file to make them permanent.
</Warning>

## Environment variables

OpenBB automatically reads any environment variable whose lowercased name matches a known credential key:

```bash
export FMP_API_KEY="YOUR_FMP_KEY"
export FRED_API_KEY="YOUR_FRED_KEY"
export INTRINIO_API_KEY="YOUR_INTRINIO_KEY"
export TIINGO_TOKEN="YOUR_TIINGO_TOKEN"
```

Environment variables take precedence over values in `user_settings.json`. OpenBB also picks up any environment variable ending in `API_KEY`, normalizing its name to lowercase for lookup.

<Tip>
  For CI/CD pipelines and containerized deployments, environment variables are the recommended way to inject credentials without committing secrets to version control.
</Tip>

## How credential naming works

When a provider declares `credentials=["api_key"]`, the `Provider` class prepends the provider name automatically:

```python
Provider(
    name="fmp",
    credentials=["api_key"],  # becomes "fmp_api_key"
)
```

For providers that use a non-standard suffix (like Tiingo's `token`):

```python
Provider(
    name="tiingo",
    credentials=["token"],  # becomes "tiingo_token"
)
```

This namespacing prevents conflicts between providers that use the same generic credential name.

## Checking required credentials

To see which credentials a command requires before running it, check the provider's `require_credentials` flag. By default every `Fetcher` sets `require_credentials = True`. If credentials are missing when a command is executed, OpenBB raises an error before making any network requests.

```python
from openbb import obb

# Will raise if fmp_api_key is not configured
result = obb.equity.price.historical("AAPL", provider="fmp")
```

## Best practices

<AccordionGroup>
  <Accordion title="Do not commit user_settings.json to version control">
    Add `~/.openbb_platform/user_settings.json` to your `.gitignore`. For shared projects, use environment variables and a `.env` file (also gitignored) to distribute credentials among team members.
  </Accordion>
  <Accordion title="Use environment variables in production">
    For deployed applications, CI pipelines, or Docker containers, inject credentials as environment variables rather than mounting a settings file. Variable names are the uppercase form of the credential key (e.g., `FMP_API_KEY`).
  </Accordion>
  <Accordion title="Check credential status before running workflows">
    Call `obb.user.credentials` to inspect which keys are set before executing a long batch job. A missing credential will raise immediately on the first affected command rather than silently returning no data.
  </Accordion>
  <Accordion title="Rotating credentials">
    To rotate a key, update `user_settings.json` and restart your Python session, or set the new value via `obb.user.credentials.fmp_api_key = "NEW_KEY"` for the current session.
  </Accordion>
</AccordionGroup>
