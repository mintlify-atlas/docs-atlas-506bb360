This guide gets you from a fresh Python environment to querying live financial data in four steps.

<Note>
Python 3.10 or later is required. Most data in this guide uses `yfinance`, which is free and requires no API key.
</Note>

<Steps>
  <Step title="Install OpenBB">
    Install the `openbb` package from PyPI:

    ```bash
    pip install openbb
    ```

    This installs the core package along with all standard extensions (equity, crypto, economy, derivatives, fixed income, ETF, index, news, commodity, currency) and the default set of data providers.
  </Step>

  <Step title="Import and run your first query">
    Open a Python shell or notebook and run:

    ```python
    from openbb import obb

    result = obb.equity.price.historical("AAPL")
    ```

    Every OpenBB command returns an `OBBject` — a standardized container with the data and metadata attached.

    ```python
    print(result)
    # OBBject
    # id: 0197f...
    # results: [EquityHistoricalData(...), ...]
    # provider: yfinance
    # warnings: None
    # chart: None
    # extra: {'metadata': {...}}
    ```

    Convert to a pandas DataFrame with one call:

    ```python
    df = result.to_dataframe()
    print(df.head())
    #             open    high     low   close    volume
    # date
    # 2024-01-02  185.15  185.15  182.73  185.19  79009280
    # 2024-01-03  184.22  185.88  183.43  184.25  55765280
    # ...
    ```
  </Step>

  <Step title="Try more data categories">
    OpenBB covers a broad range of asset classes and economic data.

    <CodeGroup>

    ```python Fundamental data
    from openbb import obb

    # Income statement (trailing twelve months)
    income = obb.equity.fundamental.income("AAPL", period="ttm")
    df = income.to_dataframe()
    ```

    ```python Economic indicators
    from openbb import obb

    # U.S. GDP from FRED (free, no key required)
    gdp = obb.economy.indicators(symbol="GDP", provider="fred")
    df = gdp.to_dataframe()
    ```

    ```python Crypto prices
    from openbb import obb

    # Bitcoin historical price
    btc = obb.crypto.price.historical("BTCUSD", provider="yfinance")
    df = btc.to_dataframe()
    ```

    </CodeGroup>
  </Step>

  <Step title="Switch providers">
    Most endpoints support multiple providers. Pass `provider` to choose one explicitly:

    ```python
    from openbb import obb

    # Default provider (yfinance)
    result = obb.equity.price.historical("AAPL")

    # Explicit provider
    result = obb.equity.price.historical("AAPL", provider="fmp")
    ```

    The output schema stays the same regardless of which provider you use. Provider-specific fields are available under `result.results[0].model_extra` when present.

    <Tip>
    Run `obb.equity.price.historical?` in a notebook to see all available providers and parameters for any endpoint.
    </Tip>
  </Step>
</Steps>

## A note on providers and API keys

Most endpoints work out of the box using `yfinance`, which requires no API key. Some providers — including FMP, Intrinio, Tiingo, and Benzinga — require credentials for access.

To add API keys, create the file `~/.openbb_platform/user_settings.json`:

```json
{
  "credentials": {
    "fmp_api_key": "YOUR_FMP_KEY",
    "intrinio_api_key": "YOUR_INTRINIO_KEY",
    "tiingo_token": "YOUR_TIINGO_TOKEN",
    "fred_api_key": "YOUR_FRED_KEY"
  }
}
```

See [Credentials](/concepts/credentials) for the full list of provider credential keys.
